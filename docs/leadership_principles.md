## Principles

| Key                                              | Value                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| :----------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **1. Customer Obsession**                        | - **Value:** Leaders earn and keep customer trust<br>- **Project:** CR1. Designed and developed custom metrics and improved observability<br>  - **At:** Cybereason<br>  - **Situation:**<br>    - A UI server has multi-components<br>    - Difficult to troubleshoot<br>  - **Action:** Improved observability<br>  - **Result:** Reduced MTTR for enterprise customers                                                                                                                   |
| **2. Ownership**                                 | - **Value:** Leaders act on behalf of the entire company, beyond just one team<br>- **Project:** EN1. Developed a generator tool for manifests and Terraform configurations for new microservices<br>  - **At:** enechain Corporation<br>  - **Situation:**<br>    - Developers had to write manifests and Terraform configurations when creating new microservices<br>    - Slowed down the development speed<br>  - **Action:** Developed a generator tool for manifests and Terraform configuration<br>  - **Result:** Reduced time to first deploy for all teams |
| **3. Invent and simplify**                       | - **Value:** Leaders expect innovation and find ways to simplify<br>- **Project:** IT4. Migrated from a modular monolith to microservices architecture<br>  - **At:** Industry Technology<br>  - **Situation:**<br>    - Adopted a monolith architecture first<br>    - Write traffic on the database was increasing<br>  - **Action:** Migrated to microservices architecture and separated the responsibility<br>  - **Result:** Simplified the architecture and improved the database performance          |
| **4. Are right, a lot**                          | - **Value:** Leaders have strong judgment and good instincts<br>- **Project:** GU1. Analyzed data from millions of users and designed A/B testing<br>  - **At:** Gunosy<br>  - **Situation:** User retention rates were declining<br>  - **Action:** Designed A/B testing scenario (Optimizing the first view) and implemented<br>  - **Result:** Increased user retention rate beyond the standard A/B testing                                                                       |
| **5. Learn and be curious**                      | - **Value:** Leaders are never done learning and consistently seek to improve ourselves<br>- **Project:** EU1. Developed a processor that compiles user scripts into WebAssembly files and executes them efficiently on a wasm runtime<br>  - **At:** Eukarya<br>  - **Situation:**<br>    - Needed to create an extension for users to use their custom scripts<br>    - Needed to learn WebAssembly (Allows efficient execution of code written in various programming languages)<br>  - **Action:**<br>    - Researched and developed<br>    - Documented<br>  - **Result:** Provided users with useful features and built a knowledge base for the team. |
| **6. Hire and develop the best**                 | - **Value:** Leaders develop leaders and take seriously our role in coaching others<br>- **Project:** IT6. Migrated from MySQL 5.7 to MySQL 8.0<br>  - **At:** Industry Technology<br>  - **Situation:**<br>    - Needed to migrate MySQL 5.7 to MySQL 8.0<br>    - Encountered an incident<br>  - **Action:**<br>    - Resolved the incident and successfully migrated<br>    - Did pair programming with team members<br>  - **Result:** Led the project to success and built a knowledge base for the team |
| **7. Insist on the highest standards**           | - **Value:** Leaders are continually raising the bar and driving their teams to deliver high-quality products<br>- **Project:** LX1. Developed an event-driven notification system within a closed network<br>  - **At:** LayerX<br>  - **Situation:**<br>    - Needed to develop a notification system to tell the workflow status to users<br>    - Our client was Japan's top financial institution, and we had the highest security requirement<br>  - **Action:**<br>    - Communicated with the financial institution's network manager<br>    - Developed the system<br>  - **Result:** Released the system with the highest security standards |
| **8. Think big**                                 | - **Value:** Leaders create and communicate a bold direction that inspires results<br>- **Project:** IT1. Designed SLI/SLO and developed custom metrics and tracing<br>  - **At:** Industry Technology<br>  - **Situation:**<br>    - Got a chance to introduce the system to Japan's largest real estate company<br>    - But realized that the existing operation didn't meet the required level<br>  - **Action:**<br>    - Decided to sacrifice agility temporarily<br>    - Introduced SRE practices<br>  - **Result:** Got a contract with the enterprise customer |
| **9. Bias for action**                           | - **Value:** Speed matters in business. Leaders take calculated risks<br>- **Project:** EN2. Developed a system for local development by deploying Telepresence to the Kubernetes cluster<br>  - **At:** enechain Corporation<br>  - **Situation:**<br>    - Each microservice depended on the others<br>    - Slowed down the local development speed<br>  - **Action:**<br>    - Decided to introduce Telepresence (OSS to help create a local development environment)<br>    - It lowered security standards, so I limited the scope to the dev cluster and added security patches<br>  - **Result:** Minimized security risks and improved the development speed |
| **10. Frugality**                                | - **Value:** Leaders accomplish more with less<br>- **Project:** AL1. Developed a serverless training pipeline for LLMs<br>  - **At:** Algomatic<br>  - **Situation:**<br>    - Needed a training pipeline for fine-tuning LLMs<br>    - Needed costs efficient solution<br>  - **Action:**<br>    - Utilized serverless architecture<br>    - Developed a training pipeline<br>  - **Result:** Enabled fine-tuning LLMs efficiently                                                              |
| **11. Earn trust**                               | - **Value:** Leaders listen attentively, speak candidly, and treat others respectfully<br>- **Project:** 2. Analyzed thread dumps and heap dumps to troubleshoot a server<br>  - **At:** Cybereason<br>  - **Situation:**<br>    - A UI server crashed occasionally for an enterprise customer<br>    - This incident ticket involved with many teams<br>  - **Action:**<br>    - Gave instructions to other teams as an incident commander<br>    - Analyzed thread dumps and heap dumps<br>  - **Result:** Organized details about the incident response and solved the availability issue |
| **12. Dive deep**                                | - **Value:** Leaders operate at all levels, stay connected to the details<br>- **Project:** IT3. Tuned indexes and queries of databases and implemented read replica and cache<br>  - **At:** Industry Technology<br>  - **Situation:** Had frequent slow queries<br>  - **Action:**<br>    - Tuned indexes and queries of databases<br>    - Implemented read replica and cache<br>  - **Result:** Reduced the number of slow queries                                                     |
| **13. Have Backbone; Disagree and Commit (Conflicts)** | - **Value:** Leaders challenge decisions when we disagree. Once a decision is determined, we commit<br>- **Project:** (New) Migrated Google Cloud to Oracle Cloud<br>  - **At:** Cybereason<br>  - **Situation:** Executives planned to migrate from Google Cloud to Oracle Cloud<br>  - **Action:**<br>    - Disagree: Used number & vision<br>    - Commit:<br>  - **Result:** Created a better migration plan and reduced infrastructure costs                                              |
| **14. Deliver results**                          | - **Value:** Leaders deliver outcomes with the right quality and in a timely fashion<br>- **Project:** IT2. Developed a group chat service and notification system<br>  - **At:** Industry Technology<br>  - **Situation:**<br>    - Needed to develop a group chat system<br>    - Schedule was tight<br>  - **Action:**<br>    - Adopted polling (Inefficient but Simple) instead of WebSocet (Efficient but Complicated) to simplify<br>    - Developed a group chat system<br>  - **Result:** Developed the expected system in time |
| **15. Strive to be Earth's best employer**        | - **Value:** Leaders have a vision for and commitment to our employee's personal success<br>- **Project:** (New)<br>  - **At:** Industry Technology<br>  - **Situation:** One of our engineers repeatedly delayed the deadline for a feature development<br>  - **Action**<br>    - Communicated and found he was lack of understanding of the product code<br>    - Reassigned him to an important refactoring task with a more flexible deadline<br>  - **Result:** He became familiarized and started handling complex feature development tasks |
| **16. Success and scale bring broad responsibility** | - **Value:** Leaders create more than we consume and always leave things better than how we found them<br>- **Project:** IT4. Migrated from a modular monolith to microservices architecture<br>  - **At:** Industry Technology<br>  - **Situation:**<br>    - Adopted a monolith architecture first<br>    - Write traffic on the database was increasing as the business grows<br>  - **Action:** Migrated to microservices architecture and separated the responsibility<br>  - **Result:** Simplified the architecture and improved the database performance  |
